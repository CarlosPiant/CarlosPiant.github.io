This chapter creates a synthetic dataset for a binary infertility outcome generated by a logistic model. The design is inspired by the reproductive-history application studied by Trichopoulos and coauthors, who examined how prior spontaneous and induced abortions related to secondary infertility. The synthetic version here is not a replication of their retrospective case-control design. Instead, it creates a prospective-style binary-outcome dataset so that the full logistic data-generating process is known in advance. That makes it useful for testing whether a fitted logistic regression can recover the true parameters.

The purpose of the chapter is straightforward. We want to build data in a way that is transparent, reproducible, and mathematically interpretable. The same logic can later be adapted for hospital readmission, policy uptake, treatment adherence, mortality, screening participation, or any other binary outcome that appears across health economics and decision sciences.

## What variables will be created

The synthetic sample will contain the following variables.

`age` will represent age in years. `parity` will count prior births. `education` will have three levels: low, medium, and high. `spontaneous` will count prior spontaneous abortions in grouped form as 0, 1, or 2, where 2 means two or more. `induced` will do the same for prior induced abortions. `smoking` will be a binary risk factor. The outcome `infertility` will equal 1 if infertility occurs and 0 otherwise.

These variables are chosen because they give the logistic model something realistic to work with: a mix of continuous, binary, ordinal, and categorical predictors, all of which are common in applied health data.

## The data-generating process

The core assumption is that the outcome follows a Bernoulli distribution with success probability determined by a logit link:

$$
Y_i \sim \text{Bernoulli}(p_i),
$$

with

$$
\text{logit}(p_i) =
\beta_0 +
\beta_1 \text{age}_i +
\beta_2 \text{parity}_i +
\beta_3 \mathbb{1}(\text{education}_i = \text{medium}) +
\beta_4 \mathbb{1}(\text{education}_i = \text{high}) +
\beta_5 \text{spontaneous}_i +
\beta_6 \text{induced}_i +
\beta_7 \text{smoking}_i.
$$

For this simulation, the true coefficients are set to

$$
\beta_0 = -2.3,\;
\beta_1 = 0.045,\;
\beta_2 = -0.32,\;
\beta_3 = -0.28,\;
\beta_4 = -0.46,\;
\beta_5 = 1.05,\;
\beta_6 = 0.78,\;
\beta_7 = 0.42.
$$

These choices imply a substantively reasonable pattern. Older age raises risk. Higher parity lowers risk. Higher education is somewhat protective relative to the low-education group. Prior spontaneous and induced abortions increase risk, and smoking adds an additional increase.

The simulation uses a large enough sample that the fitted model should recover the true parameters reasonably well, though not perfectly. That last phrase matters. A simulation is still a sample, so the estimates should be close to the truth on average, not identical in every run.

## Step 1: Generate the synthetic sample

```r
set.seed(2026)

n <- 8000

age <- pmax(round(rnorm(n, mean = 31, sd = 5)), 20)
parity <- pmin(rpois(n, lambda = 1.8), 6)

education <- sample(
 c("low", "medium", "high"),
 size = n,
 replace = TRUE,
 prob = c(0.25, 0.45, 0.30)
)

spontaneous <- sample(
 0:2,
 size = n,
 replace = TRUE,
 prob = c(0.72, 0.20, 0.08)
)

induced <- sample(
 0:2,
 size = n,
 replace = TRUE,
 prob = c(0.68, 0.22, 0.10)
)

smoking <- rbinom(
 n,
 size = 1,
 prob = plogis(-0.6 + 0.35 * (education == "low") - 0.20 * (education == "high"))
)

low_education <- as.integer(education == "low")
high_education <- as.integer(education == "high")

eta <- -2.3 +
 0.045 * age -
 0.32 * parity +
 0.28 * low_education -
 0.18 * high_education +
 1.05 * spontaneous +
 0.78 * induced +
 0.42 * smoking

true_probability <- plogis(eta)
infertility <- rbinom(n, size = 1, prob = true_probability)

synthetic_infertility <- data.frame(
 infertility,
 age,
 parity,
 education = factor(education, levels = c("low", "medium", "high")),
 spontaneous,
 induced,
 smoking
)

simulation_summary <- data.frame(
 sample_size = nrow(synthetic_infertility),
 event_rate = mean(synthetic_infertility$infertility),
 mean_age = mean(synthetic_infertility$age),
 mean_parity = mean(synthetic_infertility$parity)
)

simulation_summary[, c("event_rate", "mean_age", "mean_parity")] <-
 round(simulation_summary[, c("event_rate", "mean_age", "mean_parity")], 3)

knitr::kable(
 simulation_summary,
 caption = "Summary of the synthetic logistic-regression dataset"
)
```

That code has done two conceptually different things. First, it created the covariates. Second, it used the logistic formula to map those covariates into probabilities, and then drew a binary outcome from a Bernoulli distribution. The probabilities are the systematic part of the model. The Bernoulli draw is the stochastic part.

## Step 2: Fit the model that matches the truth

Now estimate the same logistic specification that generated the data:

```r
logit_fit <- glm(
 infertility ~ age + parity + education + spontaneous + induced + smoking,
 data = synthetic_infertility,
 family = binomial
)

truth <- c(
 "(Intercept)" = -2.30,
 age = 0.045,
 parity = -0.32,
 educationmedium = -0.28,
 educationhigh = -0.46,
 spontaneous = 1.05,
 induced = 0.78,
 smoking = 0.42
)

comparison_table <- data.frame(
 term = names(coef(logit_fit)),
 true_value = truth[names(coef(logit_fit))],
 estimated_value = coef(logit_fit)
)

comparison_table$bias <- comparison_table$estimated_value - comparison_table$true_value
comparison_table[, c("true_value", "estimated_value", "bias")] <-
 round(comparison_table[, c("true_value", "estimated_value", "bias")], 3)

knitr::kable(
 comparison_table,
 caption = "True and estimated coefficients under the correctly specified logistic model"
)
```

This is the main reason synthetic data are useful. We know the truth because we built it. That means we can ask a sharper question than we usually can with real data: did the fitted model recover the generating parameters?

In a correctly specified model with a reasonably large sample, the answer should be approximately yes. Small discrepancies are expected because the simulated sample still contains random noise. But the fitted coefficients should line up closely with the true values.

## Step 3: Compare probabilities rather than coefficients alone

Coefficients are useful, but probabilities are often easier to interpret. The next code block compares the true and fitted probabilities across age for two reproductive-history profiles.

```r
profiles <- expand.grid(
 age = seq(22, 42, by = 1),
 spontaneous = c(0, 1),
 induced = c(0, 1)
)

profiles$parity <- 1
profiles$education <- factor("medium", levels = c("low", "medium", "high"))
profiles$smoking <- 0

profiles$true_eta <- -2.3 +
 0.045 * profiles$age -
 0.32 * profiles$parity -
 0.28 * (profiles$education == "medium") -
 0.46 * (profiles$education == "high") +
 1.05 * profiles$spontaneous +
 0.78 * profiles$induced +
 0.42 * profiles$smoking

profiles$true_probability <- plogis(profiles$true_eta)
profiles$fitted_probability <- predict(logit_fit, newdata = profiles, type = "response")

profiles$history <- paste0(
 "spont=", profiles$spontaneous,
 ", induced=", profiles$induced
)

ggplot2::ggplot(
 profiles,
 ggplot2::aes(x = age, color = history)
) +
 ggplot2::geom_line(
 ggplot2::aes(y = true_probability),
 linewidth = 1.1
 ) +
 ggplot2::geom_line(
 ggplot2::aes(y = fitted_probability),
 linetype = 2,
 linewidth = 0.9
 ) +
 ggplot2::labs(
 title = "True and fitted probabilities in the synthetic logistic dataset",
 subtitle = "Solid lines show the true probabilities; dashed lines show the fitted model",
 x = "Age",
 y = "Probability of infertility",
 color = "Profile"
 ) +
 ggplot2::scale_color_manual(values = c("#0b5d4b", "#4c566a", "#b54708", "#7a4e1d")) +
 ggplot2::theme_minimal(base_size = 12)
```

This figure is often more informative than a coefficient table alone. If the dashed lines stay close to the solid lines, the fitted model is reproducing the true probability surface well.

## Step 4: Check odds ratios

Because logistic regression is usually reported in odds-ratio form, it is often useful to compare the true and estimated odds ratios directly.

```r
or_table <- data.frame(
 term = comparison_table$term,
 true_odds_ratio = exp(comparison_table$true_value),
 estimated_odds_ratio = exp(comparison_table$estimated_value)
)

or_table[, c("true_odds_ratio", "estimated_odds_ratio")] <-
 round(or_table[, c("true_odds_ratio", "estimated_odds_ratio")], 3)

knitr::kable(
 or_table,
 caption = "True and estimated odds ratios in the synthetic dataset"
)
```

This is a useful final check because many applied papers interpret logistic models through odds ratios rather than through log-odds coefficients.

## Main assumptions behind this simulation

This synthetic dataset is only as informative as the assumptions built into it.

The first assumption is that the logit is the correct link. The second is that the relationship between the covariates and the log-odds is linear in the chosen coding. The third is that the observations are conditionally independent. The fourth is that the included variables are enough to describe the data-generating process, since here we are simulating without omitted confounding or measurement error.

Those assumptions are useful for learning, but real data are often messier. Real outcomes may have clustering, nonlinear effects, misclassification, selection bias, or missing variables. That is why simulation is valuable. It gives a clean benchmark before those complications are introduced.

## How to adapt this template

Once the basic structure is clear, the same simulation template can be modified easily. You can change the coefficient sizes, alter the event rate by moving the intercept, add interactions, add nonlinear terms, create treatment assignment mechanisms, or introduce measurement error and missingness. Each of those changes gives a new synthetic laboratory for testing methods.

In practice, simulation becomes especially useful when you want to know how an estimator behaves under a known truth. Does separation create instability? How many observations are needed before a coefficient is recovered reliably? What happens when a continuous covariate is nonlinear but the fitted model is not? These are exactly the kinds of questions synthetic data can answer.

## Further reading

For the underlying binary-regression framework, Cox remains a foundational reference. For the applied reproductive-health problem that inspired the synthetic structure here, Trichopoulos and coauthors provide the substantive example. A natural next step after this chapter is to extend the simulation by adding interactions, nonlinear effects, or clustered observations and then studying how the fitted logistic model changes under misspecification.

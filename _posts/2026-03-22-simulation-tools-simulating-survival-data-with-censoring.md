This chapter creates a synthetic survival dataset in which patients are followed after curative treatment until recurrence or censoring. The aim is to simulate not just event times, but the censoring process itself. That distinction matters because many applied time-to-event datasets are shaped by two separate clocks: the latent event clock and the follow-up clock. In oncology, hospital readmission studies, and registry-linked cohort work, patients can be censored because the study ends administratively or because they are lost to follow-up before the event occurs.

The design is motivated by adjuvant colon cancer studies in which recurrence was tracked over a finite study window and some observations were censored before the event was seen. The event-time simulation logic follows the inverse-transform approach described by Bender, Augustin, and Blettner for proportional-hazards models. The practical purpose is to build synthetic follow-up data for testing Kaplan-Meier curves, Cox models, censoring summaries, or other survival-analysis procedures under a known data-generating process.

## What data will be created and what method it is meant to test

We will generate a synthetic dataset that can be used to test a proportional-hazards survival model under independent censoring. The sample will represent patients observed after colon-cancer surgery.

The variables will be:

- `age`: age in years at baseline
- `node_positive`: indicator for higher nodal burden
- `chemo`: indicator for adjuvant chemotherapy
- `event_time`: latent time to recurrence
- `dropout_time`: time to loss to follow-up
- `admin_time`: fixed administrative study end
- `censor_time`: the earlier of dropout and administrative censoring
- `time`: observed follow-up time
- `event`: event indicator equal to 1 when recurrence is observed
- `observation_type`: whether the record ends in an observed event, administrative censoring, or dropout censoring

The final step will be to fit the Cox model that matches the true proportional-hazards structure and compare the estimated log hazard ratios with the values used to generate the data.

## The data-generating process

Let patient $i$ have hazard

$$
h_i(t) = \lambda_0 \exp(\eta_i),
$$

where

$$
\eta_i =
0.015(\text{age}_i - 60) +
0.55 \text{node\_positive}_i -
0.50 \text{chemo}_i.
$$

The baseline hazard is constant:

$$
\lambda_0 = 0.00045.
$$

Under an exponential proportional-hazards model, the latent event time can be generated by inverse transform:

$$
T_i = -\frac{\log(U_i)}{\lambda_0 \exp(\eta_i)},
$$

where $U_i \sim \text{Uniform}(0,1)$.

We introduce two censoring processes. First, administrative censoring occurs at a fixed study horizon

$$
C_i^A = \tau, \qquad \tau = 1825 \text{ days}.
$$

Second, dropout censoring is generated independently as

$$
C_i^D = -\frac{\log(V_i)}{\lambda_C},
$$

where $V_i \sim \text{Uniform}(0,1)$ and

$$
\lambda_C = 0.00022.
$$

The effective censoring time is therefore

$$
C_i = \min(C_i^A, C_i^D).
$$

The observed follow-up time and event indicator are

$$
\tilde{T}_i = \min(T_i, C_i), \qquad
\Delta_i = \mathbb{1}(T_i \le C_i).
$$

This structure is the heart of the chapter. The event process and the censoring process are simulated separately and only combined at the end. That is the main design principle analysts need when building realistic survival simulations.

## Step 1: Generate baseline covariates

``` r
set.seed(2032)

n <- 5000

age <- pmin(pmax(round(rnorm(n, mean = 63, sd = 9)), 35), 85)
node_positive <- rbinom(n, size = 1, prob = 0.42)
chemo <- rbinom(n, size = 1, prob = 0.55)

baseline_data <- data.frame(
 age = age,
 node_positive = node_positive,
 chemo = chemo
)

baseline_summary <- data.frame(
 sample_size = nrow(baseline_data),
 mean_age = mean(baseline_data$age),
 node_positive_rate = mean(baseline_data$node_positive),
 chemo_rate = mean(baseline_data$chemo)
)

baseline_summary[, c("mean_age", "node_positive_rate", "chemo_rate")] <-
 round(baseline_summary[, c("mean_age", "node_positive_rate", "chemo_rate")], 3)

knitr::kable(
 baseline_summary,
 caption = "Baseline characteristics of the synthetic survival cohort"
)
```

The covariates are simple, but they create heterogeneity in recurrence risk. Older patients and those with positive nodes have higher hazards, while chemotherapy lowers the hazard.

## Step 2: Generate event times and censoring times

``` r
lambda0 <- 0.00045
dropout_rate <- 0.00022
admin_time <- 1825

u_event <- runif(n)
u_dropout <- runif(n)

linear_predictor <- 0.015 * (age - 60) +
 0.55 * node_positive -
 0.50 * chemo

event_time <- -log(u_event) / (lambda0 * exp(linear_predictor))
dropout_time <- -log(u_dropout) / dropout_rate
censor_time <- pmin(admin_time, dropout_time)

time <- pmin(event_time, censor_time)
event <- as.integer(event_time <= censor_time)

observation_type <- ifelse(
 event == 1,
 "Observed event",
 ifelse(dropout_time < admin_time, "Dropout censoring", "Administrative censoring")
)

synthetic_survival_censoring <- data.frame(
 time = time,
 event = event,
 age = age,
 node_positive = node_positive,
 chemo = chemo,
 event_time = event_time,
 dropout_time = dropout_time,
 admin_time = admin_time,
 censor_time = censor_time,
 observation_type = observation_type
)
```

At this point the dataset contains the latent event process, the censoring process, and the observed follow-up outcome that results from taking the earlier of the two.

## Step 5: Fit the model that matches the true generating process

Now estimate the Cox proportional-hazards model implied by the data-generating process.

``` r
cox_fit <- survival::coxph(
 survival::Surv(time, event) ~ age + node_positive + chemo,
 data = synthetic_survival_censoring
)

true_log_hazard_ratios <- c(
 age = 0.015,
 node_positive = 0.55,
 chemo = -0.50
)

comparison_table <- data.frame(
 term = names(stats::coef(cox_fit)),
 true_value = true_log_hazard_ratios[names(stats::coef(cox_fit))],
 estimated_value = stats::coef(cox_fit)
)

comparison_table$bias <- comparison_table$estimated_value - comparison_table$true_value

comparison_table[, c("true_value", "estimated_value", "bias")] <-
 round(comparison_table[, c("true_value", "estimated_value", "bias")], 3)

knitr::kable(
 comparison_table,
 caption = "Recovery of the true log hazard ratios under the fitted Cox model",
 row.names = FALSE
)
```

## Main assumptions behind this simulation

This design assumes proportional hazards and independent censoring. The event process depends on age, nodal burden, and chemotherapy, while dropout and administrative censoring are generated separately. That separation is exactly what allows the Cox model to interpret censoring correctly.

The simulation also assumes a constant baseline hazard. That is a simplification. Real recurrence processes often have time-varying baseline risk. The value of the current design is not that it matches every substantive application in detail. It is that it makes the censoring mechanism transparent and therefore useful for teaching, benchmarking, and code testing.

## Further reading

Bender, Augustin, and Blettner provide a foundational practical treatment of generating survival times under proportional-hazards models. Cox remains the central reference for the proportional-hazards model itself. For the kind of finite follow-up oncology setting that motivates this chapter, the adjuvant colon cancer trials by Laurie and colleagues and by Moertel and colleagues are useful substantive anchors.

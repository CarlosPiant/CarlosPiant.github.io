This chapter creates a synthetic dataset in which patients are followed after hospital discharge until readmission or censoring. The aim is to build a time-to-event outcome from a known proportional-hazards data-generating process and then check whether a fitted survival model recovers the true hazard ratios. The design is motivated by the same survival-modeling logic that underlies much of applied health-outcomes research, where time to recurrence, readmission, death, or treatment discontinuation is more informative than a simple yes-or-no indicator.

The practical lesson is that survival data are not just event indicators with a clock attached. They are built from hazards, event times, and censoring times. A good simulation should make those ingredients explicit.

## What variables will be created

The synthetic sample will contain the following variables.

`age` will represent age in years. `chronic` will count chronic conditions. `program` will indicate enrollment in a transitional-care program. `event_time` will represent the latent time to readmission. `censor_time` will represent the time at which follow-up ends for administrative or other non-event reasons. `time` will be the observed follow-up time, equal to the smaller of the event and censoring times. `event` will equal 1 if readmission is observed and 0 if the observation is censored.

These variables recreate the basic structure of a survival dataset while remaining simple enough for the hazard logic to stay visible.

## The data-generating process

The simulation assumes an exponential proportional-hazards model. Patient `i` has hazard

$$
h_i(t) = \lambda_0 \exp(\eta_i),
$$

where

$$
\eta_i =
0.018(\text{age}_i - 65) +
0.30 \text{chronic}_i -
0.45 \text{program}_i.
$$

The baseline hazard is set to

$$
\lambda_0 = 0.0025.
$$

Under an exponential proportional-hazards model, the event time can be generated by inverse transform:

$$
T_i = -\frac{\log(U_i)}{\lambda_0 \exp(\eta_i)},
$$

where $U_i \sim \text{Uniform}(0,1)$. Censoring times are then generated independently, and the observed data are

$$
\tilde{T}_i = \min(T_i, C_i), \qquad
\Delta_i = \mathbb{1}(T_i \le C_i).
$$

## Step 1: Generate the survival sample

```r
set.seed(2026)

n <- 6000

age <- pmax(round(rnorm(n, mean = 66, sd = 10)), 40)
chronic <- pmin(rpois(n, lambda = 2.0), 6)
program <- rbinom(n, size = 1, prob = 0.45)

u_event <- runif(n)
u_censor <- runif(n)

linear_predictor <- 0.018 * (age - 65) +
 0.30 * chronic -
 0.45 * program

lambda0 <- 0.0025

event_time <- -log(u_event) / (lambda0 * exp(linear_predictor))
censor_time <- 120 + 245 * u_censor

time <- pmin(event_time, censor_time)
event <- as.integer(event_time <= censor_time)

synthetic_survival <- data.frame(
 time,
 event,
 age,
 chronic,
 program,
 event_time,
 censor_time
)

survival_summary <- data.frame(
 sample_size = nrow(synthetic_survival),
 event_rate = mean(synthetic_survival$event),
 median_followup = median(synthetic_survival$time),
 mean_age = mean(synthetic_survival$age),
 mean_chronic = mean(synthetic_survival$chronic)
)

survival_summary[, c("event_rate", "median_followup", "mean_age", "mean_chronic")] <-
 round(survival_summary[, c("event_rate", "median_followup", "mean_age", "mean_chronic")], 3)

knitr::kable(
 survival_summary,
 caption = "Summary of the synthetic survival dataset"
)
```

This code separates the two sources of uncertainty that matter most in survival settings. The event time comes from the hazard model. The censoring time comes from a different process. The observed data are a combination of the two.

## Step 2: Fit the proportional-hazards model

Now estimate the Cox model that matches the hazard structure used to create the data.

```r
cox_fit <- survival::coxph(
 survival::Surv(time, event) ~ age + chronic + program,
 data = synthetic_survival
)

truth <- c(
 age = 0.018,
 chronic = 0.30,
 program = -0.45
)

comparison_table <- data.frame(
 term = names(coef(cox_fit)),
 true_value = truth[names(coef(cox_fit))],
 estimated_value = coef(cox_fit)
)

comparison_table$bias <- comparison_table$estimated_value - comparison_table$true_value
comparison_table[, c("true_value", "estimated_value", "bias")] <-
 round(comparison_table[, c("true_value", "estimated_value", "bias")], 3)

knitr::kable(
 comparison_table,
 caption = "True and estimated log hazard ratios under the Cox model"
)
```

Because the Cox model leaves the baseline hazard unspecified, the most direct recovery check focuses on the covariate effects. Those are the quantities that define the hazard ratios.

## Step 3: Compare true and fitted survival curves

To make the simulation more interpretable, compare the true and fitted survival curves for two otherwise similar patient profiles.

```r
profile_data <- data.frame(
 age = c(65, 65),
 chronic = c(2, 2),
 program = c(0, 1)
)

profile_data$label <- c("Usual care", "Program")

surv_fit_profiles <- survival::survfit(cox_fit, newdata = profile_data)

curve_data <- data.frame(
 time = rep(surv_fit_profiles$time, times = ncol(surv_fit_profiles$surv)),
 surv = as.vector(surv_fit_profiles$surv),
 strata = rep(profile_data$label, each = length(surv_fit_profiles$time))
)

time_grid <- seq(0, 365, by = 5)

true_curve_data <- rbind(
 data.frame(
 time = time_grid,
 surv = exp(-lambda0 * exp(0.018 * (65 - 65) + 0.30 * 2 - 0.45 * 0) * time_grid),
 strata = "Usual care"
 ),
 data.frame(
 time = time_grid,
 surv = exp(-lambda0 * exp(0.018 * (65 - 65) + 0.30 * 2 - 0.45 * 1) * time_grid),
 strata = "Program"
 )
)

ggplot2::ggplot +
 ggplot2::geom_line(
 data = true_curve_data,
 ggplot2::aes(x = time, y = surv, color = strata),
 linewidth = 1.1
 ) +
 ggplot2::geom_step(
 data = curve_data,
 ggplot2::aes(x = time, y = surv, color = strata),
 linetype = 2,
 linewidth = 0.9
 ) +
 ggplot2::labs(
 title = "True and fitted survival curves in the synthetic dataset",
 subtitle = "Solid lines show the generating survival curves; dashed steps show the fitted Cox model",
 x = "Days since discharge",
 y = "Readmission-free survival",
 color = "Profile"
 ) +
 ggplot2::scale_color_manual(values = c("#7f5539", "#386641")) +
 ggplot2::theme_minimal(base_size = 12)
```

If the dashed step curves stay close to the solid curves, the fitted model is reproducing the true survival pattern well.

## Step 4: Check the hazard ratios

Hazard ratios are often easier to interpret than log hazard ratios.

```r
hazard_ratio_table <- data.frame(
 term = comparison_table$term,
 true_hazard_ratio = exp(comparison_table$true_value),
 estimated_hazard_ratio = exp(comparison_table$estimated_value)
)

hazard_ratio_table[, c("true_hazard_ratio", "estimated_hazard_ratio")] <-
 round(hazard_ratio_table[, c("true_hazard_ratio", "estimated_hazard_ratio")], 3)

knitr::kable(
 hazard_ratio_table,
 caption = "True and estimated hazard ratios in the synthetic survival dataset"
)
```

## Main assumptions behind this simulation

The main assumption is proportional hazards. In this synthetic design, the treatment program and the patient characteristics shift the hazard multiplicatively and that effect does not change over time. The simulation also assumes independent censoring, which means the censoring process does not depend on the unobserved event time once the simulated variables are given.

Because the event times were generated from an exponential baseline hazard, the true model is slightly more specific than the Cox model used for estimation. That is acceptable here because the Cox model is still correctly specified for the relative hazard structure and therefore should recover the covariate effects well.

## Further reading

Cox introduced the proportional-hazards model that still anchors much of applied survival analysis. Kaplan and Meier provide the classic nonparametric survival estimator that underlies most descriptive survival work. Together, they form a strong conceptual bridge between simulated event times and the models used on real health-outcomes data.

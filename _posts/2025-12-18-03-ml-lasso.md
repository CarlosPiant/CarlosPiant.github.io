<section id="introduction-when-you-have-too-many-predictors-and-not-enough-patience" class="level1">
<h1>1. Introduction: when you have too many predictors (and not enough patience)</h1>
<p>Sometimes your dataset looks like this:</p>
<ul>
<li>50 potential predictors,</li>
<li>200 observations,</li>
<li>Highly correlated variables,</li>
<li>Zero desire to manually decide which variables to keep.</li>
</ul>
<p>OLS will happily:</p>
<ul>
<li>Include everything,</li>
<li>Overfit like there's no tomorrow,</li>
<li>Give you a huge table of coefficients that nobody reads.</li>
</ul>
<p><strong>Lasso regression</strong> (Least Absolute Shrinkage and Selection Operator) says:</p>
<blockquote class="blockquote">
<p>"What if we add a penalty that not only shrinks coefficients, but can set some of them exactly to zero?"</p>
</blockquote>
<p>So lasso is like ridge regression, but with an extra power:</p>
<ul>
<li>It can <strong>perform variable selection</strong> by driving some coefficients to zero,</li>
<li>Leaving a simpler, more interpretable model.</li>
</ul>
<p>In this tutorial we'll:</p>
<ul>
<li>Explain the lasso penalty and how it differs from ridge,</li>
<li>Fit a lasso model in R using <code>glmnet</code>,</li>
<li>See which variables get selected,</li>
<li>Discuss strengths and limitations,</li>
<li>Explain why lasso is useful in <strong>HEOR and health policy</strong>.</li>
</ul>

</section>
<section id="foundations-what-is-lasso-regression" class="level1">
<h1>2. Foundations: what is lasso regression?</h1>
<section id="the-lasso-objective" class="level2">
<h2 class="anchored" data-anchor-id="the-lasso-objective">2.1. The lasso objective</h2>
<p>We start with the linear regression setup:</p>
<ul>
<li>Outcomes <span class="math inline">\(y_i\)</span>,</li>
<li>Predictors <span class="math inline">\(x_i\)</span> (rows of <span class="math inline">\(X\)</span>),</li>
<li>Coefficients <span class="math inline">\(\beta\)</span>.</li>
</ul>
<p>Lasso estimates <span class="math inline">\(\beta\)</span> by minimizing:</p>
<p><span class="math display">\[
\min_{\beta} \; \sum_{i=1}^n (y_i - x_i^\top \beta)^2
+ \lambda \sum_{j=1}^p |\beta_j|.
\]</span></p>
<p>Compare to ridge:</p>
<ul>
<li>Ridge uses <span class="math inline">\(L_2\)</span> penalty <span class="math inline">\(\sum \beta_j^2\)</span>,</li>
<li>Lasso uses <span class="math inline">\(L_1\)</span> penalty <span class="math inline">\(\sum |\beta_j|\)</span>.</li>
</ul>
<p>The <span class="math inline">\(L_1\)</span> penalty has a lovely property: it often produces <strong>exactly zero</strong> coefficients, effectively selecting a subset of predictors.</p>
</section>
<section id="geometric-intuition-short-version" class="level2">
<h2 class="anchored" data-anchor-id="geometric-intuition-short-version">2.2. Geometric intuition (short version)</h2>
<p>If you imagine the constraint region:</p>
<ul>
<li>Ridge: a <strong>circle (or sphere/ellipsoid)</strong> in coefficient space,</li>
<li>Lasso: a <strong>diamond (or cross-polytope)</strong>.</li>
</ul>
<p>The corners of the lasso constraint region align with axes, so:</p>
<ul>
<li>The solution often lands on a corner,</li>
<li>Some coordinates (coefficients) become exactly zero.</li>
</ul>
</section>
<section id="choosing-lambda" class="level2">
<h2 class="anchored" data-anchor-id="choosing-lambda">2.3. Choosing lambda</h2>
<p>As with ridge, we choose <span class="math inline">\(\lambda\)</span> typically via cross-validation:</p>
<ul>
<li>Small <span class="math inline">\(\lambda\)</span> → weak penalty → many non-zero coefficients,</li>
<li>Large <span class="math inline">\(\lambda\)</span> → strong penalty → more coefficients set to zero.</li>
</ul>

<section id="strengths-of-lasso-regression" class="level1">
<h1>4. Strengths of lasso regression</h1>
<ol type="1">
<li><p><strong>Performs variable selection</strong><br>
Lasso can set coefficients to zero, producing a <strong>sparse</strong> model. This is extremely useful when you have many potential predictors and want a slimmer model.</p></li>
<li><p><strong>Can improve prediction and interpretability</strong><br>
By selecting a subset of predictors, lasso can reduce variance and make results easier to interpret (especially in HEOR, where variable overload is common).</p></li>
<li><p><strong>Handles high-dimensional settings (<span class="math inline">\(p &gt; n\)</span>)</strong><br>
Lasso can still be estimated when you have more predictors than observations, as long as <span class="math inline">\(\lambda\)</span> is large enough and the model isn't too dense.</p></li>
<li><p><strong>Natural link to more complex ML methods</strong><br>
Lasso is conceptually related to many sparsity-inducing methods in ML, such as L1-regularized logistic regression or L1-penalized generalized linear models.</p></li>
</ol>
<hr>
</section>
<section id="limitations-of-lasso-regression" class="level1">
<h1>5. Limitations of lasso regression</h1>
<ol type="1">
<li><p><strong>Unstable selection with correlated predictors</strong><br>
When predictors are highly correlated, lasso may arbitrarily pick one variable and drop others, which can be unstable and hard to interpret.</p></li>
<li><p><strong>Biased coefficient estimates</strong><br>
The L1 penalty shrinks coefficients toward zero, often more aggressively than ridge. This reduces variance but introduces more bias.</p></li>
<li><p><strong>Choice of lambda again crucial</strong><br>
As with ridge, cross-validation is required to choose <span class="math inline">\(\lambda\)</span>. Different splits or criteria may produce different selected models.</p></li>
<li><p><strong>Not ideal for strongly grouped predictors</strong><br>
If you know predictors "belong together" (e.g., sets of indicators), lasso might drop some but keep others. Group lasso or elastic net may be more appropriate.</p></li>
</ol>

<section id="why-lasso-matters-for-heor-and-health-policy" class="level1">
<h1>6. Why lasso matters for HEOR and health policy</h1>
<p>Lasso is particularly attractive when:</p>
<ul>
<li>You have <strong>rich covariate sets</strong> from claims, EHR, registries, or surveys,</li>
<li>You want to build <strong>parsimonious prediction models</strong> that still capture most signal,</li>
<li>You'd like some degree of automated variable selection.</li>
</ul>
<p>Common HEOR uses include:</p>
<ol type="1">
<li><p><strong>Building risk scores with many candidates</strong><br>
For predicting re-hospitalization, treatment failure, or high-cost patients, lasso can automatically select a subset of risk factors.</p></li>
<li><p><strong>Exploring large variable spaces</strong><br>
In exploratory modeling, lasso can be a first pass to identify potentially important predictors.</p></li>
<li><p><strong>Reducing model complexity for communication</strong><br>
A model with 15 predictors is often easier to explain to clinicians, managers, or policymakers than one with 200. Lasso helps get you there.</p></li>
<li><p><strong>Input to decision-analytic models</strong><br>
Lasso-based predictions can feed into simulations, CEA models, or budget impact models, especially when you need stable predictions from many correlated variables.</p></li>
</ol>

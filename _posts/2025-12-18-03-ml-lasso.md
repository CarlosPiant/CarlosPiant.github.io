# 1. Introduction: when you have too many predictors (and not enough patience)

Sometimes your dataset looks like this:

- 50 potential predictors,
- 200 observations,
- Highly correlated variables,
- Zero desire to manually decide which variables to keep.

OLS will happily:

- Include everything,
- Overfit like there's no tomorrow,
- Give you a huge table of coefficients that nobody reads.

**Lasso regression** (Least Absolute Shrinkage and Selection Operator) says:

> "What if we add a penalty that not only shrinks coefficients, but can set some of them exactly to zero?"

So lasso is like ridge regression, but with an extra power:

- It can **perform variable selection** by driving some coefficients to zero,
- Leaving a simpler, more interpretable model.

In this tutorial we'll:

- Explain the lasso penalty and how it differs from ridge,
- Fit a lasso model in R using `glmnet`,
- See which variables get selected,
- Discuss strengths and limitations,
- Explain why lasso is useful in **HEOR and health policy**.

---

# 2. Foundations: what is lasso regression?

## 2.1. The lasso objective

We start with the linear regression setup:

- Outcomes $y_i$,
- Predictors $x_i$ (rows of $X$),
- Coefficients $\beta$.

Lasso estimates $\beta$ by minimizing:

$$
\min_{\beta} \; \sum_{i=1}^n (y_i - x_i^\top \beta)^2
+ \lambda \sum_{j=1}^p |\beta_j|.
$$

Compare to ridge:

- Ridge uses $L_2$ penalty $\sum \beta_j^2$,
- Lasso uses $L_1$ penalty $\sum |\beta_j|$.

The $L_1$ penalty has a lovely property: it often produces **exactly zero** coefficients, effectively selecting a subset of predictors.

## 2.2. Geometric intuition (short version)

If you imagine the constraint region:

- Ridge: a **circle (or sphere/ellipsoid)** in coefficient space,
- Lasso: a **diamond (or cross-polytope)**.

The corners of the lasso constraint region align with axes, so:

- The solution often lands on a corner,
- Some coordinates (coefficients) become exactly zero.

## 2.3. Choosing lambda

As with ridge, we choose $\lambda$ typically via cross-validation:

- Small $\lambda$ → weak penalty → many non-zero coefficients,
- Large $\lambda$ → strong penalty → more coefficients set to zero.

# 3. Example in R: lasso regression with glmnet

We'll again use the `mtcars` dataset:

- Outcome: mpg,
- Predictors: other car characteristics.

But now we'll focus on:

- Which variables get selected at the best $\lambda$,
- How prediction compares to OLS.

```r

set.seed(123)

data(mtcars)

y <- mtcars$mpg
X <- as.matrix(mtcars[, setdiff(names(mtcars), "mpg")])

n <- nrow(X)
train_idx <- sample(seq_len(n), size = floor(2 * n / 3))
test_idx  <- setdiff(seq_len(n), train_idx)

X_train <- X[train_idx, ]
y_train <- y[train_idx]

X_test  <- X[test_idx, ]
y_test  <- y[test_idx]

mae <- function(a, b) mean(abs(a - b))
rmse <- function(a, b) sqrt(mean((a - b)^2))
```

## 3.1. OLS baseline (for reference)

```r

ols_fit <- lm(mpg ~ ., data = mtcars[train_idx, ])
ols_pred <- predict(ols_fit, newdata = as.data.frame(X_test))

c(
  OLS_MAE  = mae(y_test, ols_pred),
  OLS_RMSE = rmse(y_test, ols_pred)
)
```

---

## 3.2. Lasso with glmnet

```r

# install.packages("glmnet") # if not installed
library(glmnet)

lasso_cv <- cv.glmnet(
  x = X_train,
  y = y_train,
  alpha = 1,   # lasso
  standardize = TRUE
)

lasso_cv$lambda.min
lasso_cv$lambda.1se
```

We'll use `lambda.min`:

```r

best_lambda <- lasso_cv$lambda.min

lasso_pred <- predict(
  lasso_cv,
  s = best_lambda,
  newx = X_test
)

c(
  Lasso_MAE  = mae(y_test, lasso_pred),
  Lasso_RMSE = rmse(y_test, lasso_pred)
)
```

Now, check the coefficients:

```r

coef_lasso <- coef(lasso_cv, s = best_lambda)
coef_lasso
```

You should see some coefficients exactly zero. Those predictors have been
"turned off" by lasso.

# 4. Strengths of lasso regression

1. **Performs variable selection**  
   Lasso can set coefficients to zero, producing a **sparse** model. This is extremely useful when you have many potential predictors and want a slimmer model.

2. **Can improve prediction and interpretability**  
   By selecting a subset of predictors, lasso can reduce variance and make results easier to interpret (especially in HEOR, where variable overload is common).

3. **Handles high-dimensional settings ($p > n$)**  
   Lasso can still be estimated when you have more predictors than observations, as long as $\lambda$ is large enough and the model isn't too dense.

4. **Natural link to more complex ML methods**  
   Lasso is conceptually related to many sparsity-inducing methods in ML, such as L1-regularized logistic regression or L1-penalized generalized linear models.

---

# 5. Limitations of lasso regression

1. **Unstable selection with correlated predictors**  
   When predictors are highly correlated, lasso may arbitrarily pick one variable and drop others, which can be unstable and hard to interpret.

2. **Biased coefficient estimates**  
   The L1 penalty shrinks coefficients toward zero, often more aggressively than ridge. This reduces variance but introduces more bias.

3. **Choice of lambda again crucial**  
   As with ridge, cross-validation is required to choose $\lambda$. Different splits or criteria may produce different selected models.

4. **Not ideal for strongly grouped predictors**  
   If you know predictors "belong together" (e.g., sets of indicators), lasso might drop some but keep others. Group lasso or elastic net may be more appropriate.

---

# 6. Why lasso matters for HEOR and health policy

Lasso is particularly attractive when:

- You have **rich covariate sets** from claims, EHR, registries, or surveys,
- You want to build **parsimonious prediction models** that still capture most signal,
- You'd like some degree of automated variable selection.

Common HEOR uses include:

1. **Building risk scores with many candidates**  
   For predicting re-hospitalization, treatment failure, or high-cost patients,
   lasso can automatically select a subset of risk factors.

2. **Exploring large variable spaces**  
   In exploratory modeling, lasso can be a first pass to identify potentially important predictors.

3. **Reducing model complexity for communication**  
   A model with 15 predictors is often easier to explain to clinicians, managers, or policymakers than one with 200. Lasso helps get you there.

4. **Input to decision-analytic models**  
   Lasso-based predictions can feed into simulations, CEA models, or budget impact models, especially when you need stable predictions from many correlated variables.

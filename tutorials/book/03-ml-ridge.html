1. Introduction: when your regression is a bit too excited

Ordinary least squares (OLS) regression is like an over-caffeinated student:

  • It tries very hard to fit the data,
  • It may chase every little wiggle,
  • Give it many correlated predictors and it can go completely off the rails.

Enter ridge regression, which is basically OLS with a seatbelt:

“You can still fit the data, but I’m going to gently punish you if your coefficients get too big.”

Ridge regression adds a penalty on the size of the coefficients. The idea:

  • Large coefficients = more “complex” model,
  • Complexity is risky when you have noisy or collinear predictors,
  • So we shrink coefficients toward zero to reduce variance and improve prediction.

In this tutorial we’ll:

  • Explain the math behind ridge regression,
  • Fit a ridge model in R using glmnet,
  • Compare it informally to OLS,
  • Discuss why ridge matters for HEOR and health policy, especially with many correlated variables,
  • Share some references for deeper dives.

2. Foundations: what is ridge regression?

2.1. The classic linear regression setup

Suppose we have:

  • Outcome \(y \in \mathbb{R}^n\),
  • Predictors \(X \in \mathbb{R}^{n \times p}\) (columns are features),
  • Coefficients \(\beta \in \mathbb{R}^p\).

In OLS, we estimate \(\beta\) by minimizing:

\[ \min_{\beta} \sum_{i=1}^n (y_i - x_i^\top \beta)^2 \quad = \quad \min_{\beta} \|y - X\beta\|_2^2. \]

2.2. Ridge regression objective

Ridge regression modifies this by adding an \(L_2\) penalty:

\[ \min_{\beta} \; \|y - X\beta\|_2^2 + \lambda \|\beta\|_2^2 = \min_{\beta} \; \sum_{i=1}^n (y_i - x_i^\top \beta)^2 + \lambda \sum_{j=1}^p \beta_j^2. \]

Where:

  • \(\lambda \ge 0\) is the regularization parameter,
  • Larger \(\lambda\) → stronger penalty → more shrinkage.

Key properties:

  • When \(\lambda = 0\), we get OLS,
  • As \(\lambda \to \infty\), coefficients shrink toward zero,
  • Ridge never sets coefficients exactly to zero (no variable selection),
  • It stabilizes estimation in the presence of multicollinearity.

2.3. Closed-form solution

Ridge has a neat closed-form estimator:

\[ \hat{\beta}^{\text{ridge}} = (X^\top X + \lambda I_p)^{-1} X^\top y, \]

where \(I_p\) is the \(p \times p\) identity matrix.

Compared to OLS:

  • OLS: \((X^\top X)^{-1} X^\top y\),
  • Ridge: adds \(\lambda I_p\) to \(X^\top X\) to ensure invertibility and shrinkage.

This is especially helpful when:

  • \(p\) is large relative to \(n\),
  • \(X^\top X\) is nearly singular due to strong collinearity.

We’ll use the classic mtcars dataset:

  • Outcome: miles per gallon (mpg),
  • Predictors: other car characteristics (weight, horsepower, cylinders, etc.).

We’ll:

  • Split into training and test sets,
  • Fit OLS and ridge regression,
  • Compare prediction performance.

3.2. Ridge regression with glmnet

We’ll use the glmnet package, which:

  • Requires the predictors as a matrix (x) and outcome as a vector (y),
  • Uses alpha = 0 for ridge regression,
  • Uses cross-validation to pick \(\lambda\).
Code 
# install.packages("glmnet") # if not installed
library(glmnet)

# glmnet expects x matrix and y vector
ridge_cv <- cv.glmnet(
  x = X_train,
  y = y_train,
  alpha = 0,   # ridge
  standardize = TRUE
)

ridge_cv$lambda.min
[1] 3.755852
Code 
ridge_cv$lambda.1se
[1] 15.16244

We’ll use the \(\lambda\) that minimizes cross-validated error (lambda.min):

Code 
best_lambda <- ridge_cv$lambda.min

ridge_pred <- predict(
  ridge_cv,
  s = best_lambda,
  newx = X_test
)

c(
  Ridge_MAE  = mae(y_test, ridge_pred),
  Ridge_RMSE = rmse(y_test, ridge_pred)
)

4. Strengths of ridge regression

  1. Tames multicollinearity
    Ridge is very helpful when predictors are highly correlated (common in HEOR datasets with overlapping measures of risk, utilization, and socio-demographics).

  2. Improves prediction stability
    By shrinking coefficients, ridge can reduce variance and improve out-of-sample performance, especially when \(p\) is large relative to \(n\).

  3. Simple and fast to compute
    Efficient algorithms exist (glmnet, etc.), and the mathematics is clean. It’s easy to integrate into pipelines and cross-validation schemes.

  4. Plays nicely with many ML ideas
    Ridge appears in many guises (e.g., weight decay in neural nets) and provides a natural bridge from classical regression to regularized ML.

5. Limitations of ridge regression

  1. No variable selection
    Ridge shrinks coefficients, but almost never to exactly zero. If you want a sparse model (few non-zero coefficients), ridge is not ideal.

  2. Choice of lambda is crucial
    Too small → little regularization; too large → underfitting. While cross-validation helps, it adds complexity and computational cost.

  3. Interpretation changes
    Coefficients are shrunk and depend on the scale of predictors. Interpretability is trickier if variables aren’t standardized and if \(\lambda\) is large.

  4. Not magic for bad data
    Ridge helps with collinearity and overfitting, but won’t fix measurement error, omitted-variable bias, or poor causal identification.

6. Why ridge regression matters for HEOR and health policy

Ridge regression is particularly useful when:

  • You have many correlated predictors:
    • Comorbidity indicators,
    • Utilization measures,
    • Socioeconomic variables,
    • Prescription histories.
  • You care about predicting costs, utilization, or risk accurately more than about pinning down one “true” causal effect.

Use cases include:

  1. Risk adjustment models
    Predicting expected costs or utilization to adjust reimbursements or compare providers. Ridge can stabilize estimates with many overlapping risk factors.

  2. Prognostic risk scores
    Estimating hospitalization risk, mortality risk, or disease progression risk using a large pool of covariates.

  3. Feature-rich observational datasets
    Large claims or EHR datasets often have many correlated codes and measures. Ridge helps control overfitting while using many predictors.

  4. Input to downstream decision models
    Predictions from ridge models (e.g., expected costs, risks) can feed into cost-effectiveness models, budget impact analysis, or simulations.

7. Further reading

  1. Hastie, Tibshirani, & Friedman – The Elements of Statistical Learning (Ch. 3, 7)
    Classic treatment of linear models, ridge regression, and shrinkage methods.

  2. James, Witten, Hastie, & Tibshirani – An Introduction to Statistical Learning (ISL, Ch. 6)
    Very accessible introduction to ridge and lasso with R examples.

  3. Hoerl & Kennard (1970). Ridge Regression: Biased Estimation for Nonorthogonal Problems. Technometrics.
    The original ridge regression paper.

  4. glmnet documentation and vignettes
    Practical details on fitting ridge (and lasso, elastic net) in R with cross-validation and extensions.
